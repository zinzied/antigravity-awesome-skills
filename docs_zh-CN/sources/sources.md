## 贡献指南

### 如何贡献新技能

1. **检查现有技能**：确保您的技能与现有技能不重复
2. **遵循质量标准**：参考[质量标准](../quality-bar.md)
3. **使用技能模板**：参考[技能模板](../contributors/skill-template.md)
4. **明确来源**：如果基于现有工作，请明确引用
5. **验证许可证**：确保许可证兼容性

### 许可证兼容性

**接受的外部许可证：**
- MIT（最兼容）
- Apache 2.0（高度兼容）
- BSD 2/3条款（兼容）
- CC-BY/CC-BY-SA（文档友好）

**需要审核的许可证：**
- GPL/LGPL（需要特别考虑）
- 专有许可证（需要明确授权）
- 自定义许可证（需要法律审核）

### 归属要求

**必需的归属信息：**
- 原始作者姓名/用户名
- 原始许可证
- 原始仓库链接
- 修改说明（如果有）

**归属格式示例：**
```markdown
## 来源

- **原始作者**：@username
- **原始仓库**：[GitHub链接]
- **许可证**：MIT
- **适配说明**：为代理使用进行了优化
```
